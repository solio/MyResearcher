# 思考过程 - 20260501 迭代五

## 问题反馈

根据对 `20260430_105447` 输出的检查，发现以下问题：

1. **还有垃圾信息混在里面** - 比如导航、版权等模板内容仍然存在
2. **出现乱码** - 有一些无法识别的奇怪字符
3. **新闻条目不足** - 过滤后剩下的有效文章太少

## 改进方案

### 1. 增强内容清理能力 (content_cleaner.py)

**新增检测功能：**

- **乱码检测** (`is_gibberish`)：
  - 检测大量连续非ASCII字符
  - 检测大量非中英文数字标点的字符

- **低质量内容检测** (`is_low_quality`)：
  - 检测纯数字加标点的内容
  - 检测基本没有有效文字的内容
  - 检测中文字符比例过低的内容 (< 10%)

- **改进过滤逻辑**：
  - 过滤内容长度提高到 40 字符
  - 标题也需要检测是否是模板内容
  - 段落最小长度提高到 15 字符

### 2. 优化搜索策略 (searcher.py)

**多层搜索策略：**

1. **第一层搜索** - 带时间限制，请求更多结果 (max_results * 2)
2. **第二层 fallback** - 如果结果 < max_results/2，放宽时间限制重新搜索
3. **第三层 fallback** - 如果仍然不足，简化搜索词再搜索一次

**更多查询词：**
- 股票搜索从 3 个 query 增加到 5 个
- 论坛搜索从 3 个 query 增加到 5 个
- 行业搜索同样增加查询词数量

### 3. 结果不足的标记处理

**当所有搜索都无法获取足够新闻时：**
- 在 `search_stock_news` 和 `search_industry_news` 中添加警告标记条目
- 标记内容包含可能的原因说明
- 在 researcher.py 中检测这种标记，跳过情绪分析等处理
- 在 Markdown 报告中用 ⚠️ 特殊样式显示

### 4. researcher.py 适配

- 检测 `is_warning` 标记
- 有警告时跳过情绪分析和 LLM 深度分析
- Markdown 显示时单独处理警告样式
- 修复 StockSearcher 初始化参数
