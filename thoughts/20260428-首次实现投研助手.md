## 二、需要改进的地方（重点高亮，方便重新调整 prompt）

### 🔴 问题 1: LLM 提示词过于简单

**位置**: `llm.py` 中的 `analyze_news` 方法

**问题描述**:
```python
prompt = f"""
你是一个专业的价值投资分析师。请分析以下新闻，给出：
1. 热点话题总结
2. 对投资的影响分析
3. 关键信息提炼

新闻列表：
{news_text}
"""
```

**为什么不好**:
- 提示词太笼统，没有针对价值投资的具体分析框架（PE、PB、ROE 等）
- 没有区分"利好"、"利空"、"中性"信息
- 缺少对新闻真实性/可信度的判断
- 没有要求给出具体的投资建议或风险提示

**改进方向**:
应该设计一个更结构化的价值投资分析提示词，包含：
- 行业竞争格局分析
- 公司基本面变化
- 估值影响
- 风险提示
-  actionable 的建议

---

### 🔴 问题 2: 搜索关键词策略单一

**位置**: `searcher.py` 中的 `search_stock_news` 和 `search_industry_news`

**问题描述**:
```python
query = f"{stock_name} {stock_code} 最新新闻 热门话题"
```

**为什么不好**:
- 只用一个搜索 query，可能漏掉重要信息
- 没有区分"新闻"、"公告"、"研报"、"论坛讨论"等不同类型
- 缺少时间范围限定（比如"最近24小时"）
- 没有针对负面新闻/风险事件的专门搜索

**改进方向**:
- 多个 query 组合搜索（新闻+公告+研报+论坛）
- 添加时间过滤
- 增加"风险"、"利空"等反向关键词搜索

---

### 🟡 问题 3: 日期提取过于简单

**位置**: `searcher.py` 中的 `_extract_date` 方法

**问题描述**:
```python
def _extract_date(self, content: str) -> str:
    # 这里可以扩展日期提取逻辑
    # 暂时返回当前日期
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")
```

**为什么不好**:
- 直接返回当前时间，新闻的实际发布时间丢失
- 影响新闻的时效性判断

**改进方向**:
- 用正则从新闻内容中提取日期
- 从 URL/metadata 中获取时间
- 调用 LLM 来识别时间

---

### 🟡 问题 4: 缺少结果去重和排序

**问题描述**:
多个搜索源可能返回相同新闻，当前没有去重；也没有按重要性/时效性排序。

**改进方向**:
- URL 去重
- 标题相似度去重
- 按时间倒序排列
- 按"重要性"打分排序（可配置规则）

---

### 🟡 问题 5: 错误处理较弱

**问题描述**:
API 调用失败时只是打印错误并返回空数据，没有重试机制，也没有降级策略。

**改进方向**:
- 增加请求重试（retry）
- 超时处理优化
- 失败时记录日志，不影响其他股票/行业的研究
- 可以考虑缓存上一次的成功结果

---

### 🟢 问题 6: 缺少增量更新机制

**问题描述**:
每次都是全量重新搜索，没有记录已看过的新闻，可能产生重复内容。

**改进方向**:
- 维护一个已处理新闻的数据库/文件
- 只推送新出现的新闻
- 跟踪热点话题的演变

```
1. ...（原因 + 影响程度：高/中/低）

- 当前估值分位（如果有数据）
- 事件对估值的可能影响

## 五、风险提示
1. ...
2. ...

## 六、行动建议
- 建议仓位：维持/增持/减持/观望

【输出要求】
- 语言精炼，避免空话
- 区分"事实"和"观点"

```
