## 🔴 需要注意/改进的地方

### 1. Tavily 搜索结果中缺少结构化数据

**位置**：emotion.py 中的 _extract_number()

**问题描述**：
```python
def _extract_number(self, text: str, keywords: List[str]) -> int:
    # TODO: 更智能的提取逻辑
    # 暂时返回0，因为Tavily搜索结果不包含结构化的回复/点赞数据
    return 0
```

**为什么不好**：
- 无法从搜索结果中真正提取到回帖数和点赞数
- 导致帖子分类和情绪值计算都不准确
- 大部分帖子可能都被分到"股吧普通帖"

**改进方向**：
- **方案A**：使用专门的爬虫爬取雪球/股吧（需要处理反爬、登录等）
- **方案B**：换用其他支持获取论坛帖子互动数据的搜索API
- **方案C**：在提示词中让LLM从帖子内容/标题中"猜测"热度

### 2. 情绪值计算依赖互动数据，但互动数据缺失

**位置**：emotion.py 中的 calculate_emotion_score()

**问题描述**：
- 当前公式依赖 reply_count 和 like_count，但都是0
- 情绪值计算可能没有意义

**改进方向**：
- 在互动数据缺失时，可以只基于LLM对每个帖子的情绪评分（post.emotion_score）简单平均
- 或者按帖子类型给固定权重（爆值帖权重高，普通帖权重低）

### 3. 股票列表格式解析需要更健壮

**位置**：config.py 中的 _parse_stock_list()

**问题描述**：
- 支持多格式是好事，但用户可能输入错误格式
- 市值默认100亿可能不合理

**改进方向**：
- 增加格式校验和错误提示
- 可以支持从API获取实时市值（需要额外数据源）

### 4. LLM提示词可以更精准

**位置**：llm.py 中的 analyze_post_emotion()

**问题描述**：
- 当前提示词比较简单
- 可能返回-1到1之外的值，或者返回非数字内容

**改进方向**：
- 增加few-shot示例
- 更明确的输出要求
- 可以让LLM同时输出理由，便于调试

## 🟡 设计亮点

### 1. 情绪参数动态调整机制

**位置**：emotion.py 中的 StockEmotionParams.check_param_update()

**设计思路**：
- 阈值基于市值初始化：市值/100亿 * 基数
- 每5天检查一次，如果平均热度超过阈值50%则向上调整，反之向下
- 最低阈值不低于1

### 2. 可配置化权重

**位置**：config.py

**设计思路**：
- 雪球值：0.5
- 股吧热度：0.2
- 股吧爆值：0.2
- 股吧普值：0.1
- 所有参数都可以在.env中调整

### 3. 双重参数建议

**位置**：researcher.py

**设计思路**：
- 自动调整建议：基于历史数据的规则判断
- LLM调整建议：让LLM基于历史数据给出调整建议
- 两者都在报告中展示，供用户参考
