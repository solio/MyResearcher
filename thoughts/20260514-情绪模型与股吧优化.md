# 20260514 - 情绪模型与股吧优化改进思路

## 问题分析

### 原有问题
1. **情绪值计算问题**
   - 市值乘法导致高市值股票情绪值天然偏高，小盘股与大盘股不可比
   - 互动数为0时情绪值恒为0，没有区分度
   - 梯度跨度太大，归一化不足

2. **股吧数据问题**
   - 之前的解析没有正确处理股吧5列表格结构
   - datetime对象无法JSON序列化导致保存失败
   - "万"单位的阅读数没有正确解析

3. **垃圾数据问题**
   - SEO垃圾页面多
   - 行情页面、列表页面需要过滤
   - 文不对题的内容需要过滤

## 改进思路

### 1. 情绪模型改进（ASEN模型）

**核心思路**：
- 去除市值直接乘法，使用对数归一化
- 引入行业基准和市场基准做标准化
- 对零互动添加微小噪声保持区分度
- 使用tanh函数将最终值压缩在[-1,1]

**具体公式**：
```
互动强度 = log(1 + 点赞数) * α + log(1 + 回复数) * (1 - α)
归一化强度 = Z-score(互动强度, 行业历史均值, 行业历史标准差)
情绪贡献 = 情感倾向 * 归一化强度 * 平台权重 * 时间衰减
最终情绪 = tanh(Σ(情绪贡献) / 市场平均贡献)
```

**优势**：
- 去除市值偏差，跨股票可比
- 对数压缩解决长尾问题
- tanh使输出范围稳定在[-1,1]
- 零互动添加噪声避免恒为0

### 2. 股吧爬虫优化

**解析结构**：
- 股吧是标准5列表格：阅读|评论|标题|作者|最后更新
- 分别提取每一列，标题列中提取链接
- 正确解析"万"单位：如"1.2万"→12000

**时间处理**：
- 解析后的datetime对象在保存前转为字符串
- 格式："YYYY-MM-DD HH:MM:SS"

**爬取策略**：
- 爬取10页确保获取足够帖子
- 去重后筛选80条进行情绪分析
- 区分热度帖、爆值帖、普通帖

### 3. URL与内容过滤

**多层过滤**：
1. 强黑名单URL：直接过滤已知垃圾域名
2. 模式过滤：行情页、列表页、模板页特征
3. 内容检查：模板特征、导航特征、行情特征
4. 有效内容验证：检查是否有实质内容

**例外机制**：
- PDF研报保留
- 具体公告详情页保留
- 具体新闻/帖子详情页保留
