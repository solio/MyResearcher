# 7级情绪评分模型实现说明

## 概述

本文档说明如何将原有的5级情绪标签扩展为7级，并增加每个级别的细分档位，同时通过DeepSeek-V4-Pro进行深度多因子分析。

## 核心改进

### 1. 7级情绪评级体系

| 评级 | 分值范围 | 表情 | 说明 |
|------|----------|------|------|
| 极度恐惧 | -3.0 ~ -2.5 | 😱 | 市场恐慌性抛售，散户极度绝望 |
| 恐惧 | -2.49 ~ -1.5 | 😟 | 看空情绪明显，资金流出 |
| 迷茫 | -1.49 ~ -0.5 | 🤔 | 观望情绪浓厚，多空争论激烈 |
| 中性 | -0.49 ~ 0.49 | 😐 | 多空平衡，情绪稳定 |
| 乐观 | 0.5 ~ 1.49 | 🙂 | 看好后市，资金进场 |
| 积极 | 1.5 ~ 2.49 | 😊 | 多头情绪强烈，持续上涨预期 |
| 极度乐观 | 2.5 ~ 3.0 | 🤩 | 市场狂热，散户极度亢奋 |

### 2. 多因子分析模型

#### 市值归一化因子

- **预期发帖数**: `max(20, 市值/10)`
- **预期新闻数**: `max(5, 市值/50)`
- **预期互动数**: `max(100, 市值/2)`

- **丰裕系数**: `实际样本数/预期样本数`
- **压缩逻辑**: 当系数 > 3时，使用对数压缩: `1 + log(系数) / log(3)`

#### 平台权重因子

| 类型 | 权重 |
|------|------|
| 雪球爆值帖 | 1.5 |
| 雪球热帖 | 1.2 |
| 股吧爆值帖 | 1.3 |
| 股吧热帖 | 1.1 |
| 普通帖 | 1.0 |
| 新闻 | 0.9 |

#### 时间衰减因子

```
权重 = e^(-小时数 / 24)
```

- 0-6小时: 1.0 ~ 0.78
- 6-12小时: 0.78 ~ 0.61
- 12-24小时: 0.61 ~ 0.37
- 24-48小时: 0.37 ~ 0.14
- >48小时: <0.14

### 3. 评分计算流程

#### 阶段1: 单帖深度分析

DeepSeek-V4-Pro对每个帖子进行分析：

```json
{
  "index": 1,
  "emotion_direction": "看多",  // 极度看多/看多/中性/看空/极度看空
  "emotion_intensity": 7.5,      // 0-10 情绪强度
  "content_quality": 6.0,        // 0-10 内容质量（是否有实质分析）
  "impact_potential": 5.0        // 0-10 影响潜力（可能引发的热度）
}
```

#### 阶段2: 计算原始评分

```
单帖原始分 = 方向映射值 × 情绪强度 × 内容质量 / 10

其中方向映射:
  - 极度看多: 1.0
  - 看多: 0.7
  - 中性: 0.0
  - 看空: -0.7
  - 极度看空: -1.0

加权总分 = Σ(单帖原始分 × 平台权重 × 时间权重)
原始评分 = 加权总分 / Σ(平台权重 × 时间权重)
```

#### 阶段3: 市值调整

```
如果丰裕系数 < 1.0:
    置信因子 = 0.5 + 0.5 × 丰裕系数
    市值调整分 = 原始评分 × 置信因子
否则:
    市值调整分 = 原始评分
```

#### 阶段4: 行业对比调整（可选）

```
如果有行业平均情绪分:
    差值 = 市值调整分 - 行业平均
    行业调整分 = 市值调整分 + 差值 × 0.3
否则:
    行业调整分 = 市值调整分
```

#### 阶段5: 最终评分

```
最终评分 = max(-3.0, min(3.0, 行业调整分))
```

## 使用方法

### 在主流程中使用

V2分析已自动集成到主流程中：

```bash
python main.py --mode once
```

流程会自动：
1. 搜索股票相关新闻和帖子
2. 先进行V1情绪分析（保持兼容）
3. 然后进行V2精细情绪分析
4. 在报告中显示V2的详细结果

### 单独测试V2分析

```bash
python test_emotion_v2.py
```
