## 🔴 问题分析与说明

### 为什么之前热度帖都是0？

**根本原因：** Tavily搜索结果中不包含结构化的回帖数/点赞数字段。

**Tavily返回的搜索结果格式：**
```json
{
  "title": "帖子标题",
  "url": "https://...",
  "content": "摘要内容...",
  "source": "来源网站"
}
```
- 没有 `reply_count` 或 `like_count` 字段
- content里可能包含互动信息，但不保证且格式不统一

### 当前解决方案：

1. **尽力提取：** 用正则从标题和内容中尝试提取数字
2. **智能模拟：** 在提取不到时，根据市值给一个合理的模拟值（仅用于演示让分类逻辑能工作）
3. **记录日志：** 方便调试看到哪些帖子的数据是模拟的

### 真正的解决方案（后续改进）：

**方案A：直接爬取雪球/股吧**
- 使用requests+BeautifulSoup直接爬取网页
- 可以获取完整的表格数据（阅读/评论/标题/作者/更新）
- 需要处理反爬、分页等问题

**方案B：使用更专业的财经数据API**
- 东方财富Choice、Wind等
- 提供结构化的论坛热度数据
- 需要付费

**方案C：使用其他搜索API**
- 例如Serper.dev（更便宜）
- 或Google Custom Search JSON API
- 可能提供更丰富的数据

当前：
```

```

可以改进为：
```
请分析以下N个帖子的情绪，每个帖子返回一个-1到1之间的数字：
-1 = 极度悲观/强烈利空
-0.5 = 偏悲观/利空
0 = 中性
0.5 = 偏乐观/利好
1 = 极度乐观/强烈利好

请严格按格式返回，每行一个数字，格式为：
帖子编号: 情绪值
例如：
0: 0.5
1: -0.2
2: 0.0
```
