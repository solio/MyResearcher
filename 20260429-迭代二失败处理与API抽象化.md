## 🔴 高亮：本次改进的重点

### 1. 搜索 API 可替换

**位置**: `searcher.py`

**改进点**:
- 定义了 `BaseSearchProvider` 抽象基类
- `TavilySearchProvider` 是一个实现
- `StockSearcher` 接受任何 `BaseSearchProvider` 实例

**为什么这样做**:
- 未来可以轻松切换到其他搜索 API（如 Google Search、Serper 等）
- 可以用 MockProvider 做测试
- 符合"依赖倒置"原则

### 2. LLM API 可替换

**位置**: `llm.py`

**改进点**:
- 定义了 `BaseLLMProvider` 抽象基类
- `DeepSeekLLMProvider` 是一个实现
- `StockAnalyzer` 接受任何 `BaseLLMProvider` 实例

**为什么这样做**:
- 未来可以切换到 GPT-4、Claude、文心一言等
- 可以用 MockProvider 做测试

### 3. 重试机制 + 指数退避

**位置**: `searcher.py` 和 `llm.py`

**实现**:
```python
for attempt in range(1, self.max_retries + 1):
    try:
        ...
    except Exception as e:
        if attempt < self.max_retries:
            wait_time = 2 ** attempt  # 指数退避: 2, 4, 8...
            time.sleep(wait_time)
```

**为什么这样做**:
- 临时网络问题不需要立即放弃
- 指数退避避免风暴

### 4. 终端错误高亮

**位置**: `console.py`

**改进点**:
- 红色显示 Payment Required（需要付费）
- 红色显示 Unauthorized（认证失败）
- 黄色显示 Rate Limit（频率超限）
- 黄色显示 Timeout（超时）

**为什么这样做**:
- 用户一眼就能看出问题类型
- 体验更好

## 🟡 待改进的地方

### 1. 文件名历史对比的策略

**位置**: `researcher.py` 的 `_find_latest_data_file`

**当前实现**:
- 找当天目录下所有 `*数据*.json` 文件，按名称排序取最新

**问题**:
- 文件名依赖 `YYYYMMDD_HHMMSS` 格式
- 如果有其他文件混入可能出错

**改进空间**:
- 可以用文件修改时间代替文件名排序
- 或者用更严格的正则匹配

### 2. 摘要生成失败时的处理

**位置**: `researcher.py`

**当前实现**:
- 如果 `generate_summary` 返回 None，用 "摘要生成失败"
- 然后对比时可能因为不相似而继续分析

**问题**:
- 摘要失败可能导致每次都认为"有更新"，浪费 Token

**改进空间**:
- 摘要失败时，直接用新闻标题拼接成一个简单摘要
- 或者标记为"无法对比"，默认继续分析

### 3. prompt 可以根据新闻数量动态调整

**位置**: `llm.py`

**问题**:
- 如果新闻很多，一次性放进去可能超 Token
- 如果新闻很少，又显得浪费

**改进空间**:
- 先统计 token 数量
- 如果太多，只取前 N 条新闻，或截断每条的 content
- 可以配置最大输入 token 数
