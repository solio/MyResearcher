## 🔴 问题 1: 论坛搜索不够精准

**位置**：`searcher.py` 的 `search_stock_news` 和 `search_industry_news`

**当前实现**：
```python
query = f"{stock_name} {stock_code} 雪球 股吧 论坛讨论 投资者情绪"
```

**问题描述**：
- 只是把"雪球"、"股吧"当关键词，Tavily 可能搜不到精准的帖子
- 没有区分不同论坛的特点
- 没有针对热门帖子的筛选条件

**为什么不好**：
- 可能返回的只是新闻网站提到"雪球"的文章，而不是真正的雪球帖子
- 没有筛选浏览量/回复数高的热门讨论
- 情绪分析的输入质量直接受影响

**改进方向**（下次迭代）：
```
# 可以拆分成多个 query
query1 = f"site:xueqiu.com {stock_name} {stock_code}"
query2 = f"site:guba.eastmoney.com {stock_name} {stock_code}"
```
或者：
- 考虑直接爬取（如果允许）
- 用 Tavily 的 include_domains 参数指定站点

## 🟡 问题 2: 历史对比的相似度算法简单

**位置**：`researcher.py` 的 `HistoryManager.is_similar_content`

**当前实现**：
```python
similarity = SequenceMatcher(None, summary1, summary2).ratio()
return similarity >= threshold
```

**问题描述**：
- 直接比较字符串，对语义理解不够
- 可能出现：文字不同但意思相同 → 误判为不相似
- 也可能出现：文字相似但重点不同 → 误判为相似

**为什么不好**：
- 浪费 Token：明明内容更新了，却误判为相似 → 跳过了
- 或相反：明明内容没更新，却误判为不同 → 浪费 Token 重新分析

**改进方向**（下次迭代可选）：
- 用 LLM 做摘要的相似度判断（需要消耗 Token）
- 或提取关键词集合，用 Jaccard 相似度
- 或比较新闻 URL 集合的重合度（更简单可靠）
